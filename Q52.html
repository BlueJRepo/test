A    The time that is allowed to pass between commiting the transaction and the completion of the mule flow.
            After timeout, flow processing triggers an error.

B    The time that is allowed to pass between receiving JMS messages on the same JMS connection.
            After the timeout, a new JMS connection is established.

C    The time that is allowed to pass for stale JMS consumer threads to be destroyed.
            After timeout, new JMS consumer thread is created.

D    The time that is allowed to pass without the transaction being ended explicitly.
            After timeout the transaction is fully rolled back.
